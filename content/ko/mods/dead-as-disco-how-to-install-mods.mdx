## Dead as Disco 모딩을 위한 전제 조건

게임 디렉토리의 파일 구조를 살펴보기 전에, 적절한 도구와 계정이 준비되어 있는지 확인해야 합니다. 2026년의 모딩은 더욱 간소화되었지만, 파일 손상이나 게임 충돌을 방지하기 위해 기초적인 단계를 따르는 것이 중요합니다.

이러한 모드들의 주요 허브는 Nexus Mods입니다. 대부분의 제작자는 플랫폼의 보안과 사용 편의성 때문에 그곳에 파일을 업로드합니다. 몇 킬로바이트 이상의 파일을 다운로드하려면 무료 계정이 필요합니다. 또한, 거의 모든 모드가 압축된 형식으로 배포되므로 WinRAR, 7-Zip 또는 Windows 내장 압축 해제 도구와 같은 유틸리티가 있는지 확인하세요.

| 요구 사항 | 목적 | 권장 사항 |
| :--- | :--- | :--- |
| **Nexus Mods 계정** | 모드 파일 액세스 및 다운로드 | 무료 계정으로 충분함 |
| **압축 해제 소프트웨어** | .zip, .rar 또는 .7z 파일 열기 | **7-Zip** 또는 **WinRAR** |
| **Steam 버전** | 파일 액세스를 위한 기본 게임 플랫폼 | 2026년 빌드로 업데이트됨 |
| **디스크 공간** | 모드 백업 및 새 스킨 저장 | 최소 500MB 추가 공간 |

> **경고:** 모드를 설치하기 전에 항상 원본 세이브 파일을 백업하세요. 스킨 모드는 일반적으로 안전하지만, 게임 파일을 수정하면 업데이트 중에 예기치 않은 동작이 발생할 수 있습니다.

## Dead as Disco 모드 설치 방법: 단계별 안내

이 게임의 모드 설치에는 언리얼 엔진 기반 게임에서 흔히 볼 수 있는 특정 디렉토리 구조가 필요합니다. 이 과정에서는 게임 엔진이 기본 에셋보다 우선순위로 인식하는 "하위 폴더"를 만들어야 합니다. 새로운 스킨이 게임에 나타나도록 다음 단계를 주의 깊게 따르세요.

### 1단계: 원하는 모드 다운로드
Nexus Mods의 Dead as Disco 섹션으로 이동합니다. 예를 들어 인기 있는 '스파이더맨 어드밴스드 슈트(Spider-Man Advanced Suit)'와 같은 스킨을 선택하고 "Manual Download"를 클릭합니다. "Slow Download" 옵션을 선택하고(프리미엄 계정이 없는 경우) 아카이브 다운로드가 완료될 때까지 기다립니다.

### 2단계: 모드 파일 압축 해제
"다운로드" 디렉토리에서 다운로드한 폴더를 찾습니다. 대개 압축 파일 형태일 것입니다. 파일을 마우스 오른쪽 버튼으로 클릭하고 "모두 압축 풀기" 또는 "[폴더 이름]에 압축 풀기"를 선택합니다. 이렇게 하면 실제 모드 파일이 나타나며, 일반적으로 `.pak`, `.ucas`, `.utoc`의 세 가지 파일 유형으로 구성됩니다.

### 3단계: 게임 디렉토리 찾기
파일을 설치하려면 Steam이 하드 드라이브의 어디에 게임을 설치했는지 찾아야 합니다.
1. **Steam 라이브러리**를 엽니다.
2. **Dead as Disco**를 마우스 오른쪽 버튼으로 클릭합니다.
3. **관리** > **로컬 파일 탐색**을 선택합니다.
4. 그러면 게임 루트 폴더가 표시된 창이 열립니다.

### 4단계: 모드 폴더 생성
로컬 파일 내에서 콘텐츠 구조를 더 깊게 파고들어야 합니다. 다음 경로를 따르세요:
`Dead as Disco > Lakota > Content > Paks`

`Paks` 폴더 안에 새 폴더를 만들어야 합니다. 이 폴더는 올바르게 작동하기 위해 반드시 이름 앞에 물결표(tilde)가 있어야 합니다. 마우스 오른쪽 버튼을 클릭하고 "새 폴더"를 선택한 후 이름을 다음과 같이 지정합니다:
`~mod`

| 폴더 레벨 | 경로 이름 | 필요한 작업 |
| :--- | :--- | :--- |
| **루트** | Dead as Disco | Steam을 통해 열기 |
| **하위 폴더 1** | Lakota | 더블 클릭하여 진입 |
| **하위 폴더 2** | Content | 더블 클릭하여 진입 |
| **하위 폴더 3** | Paks | 더블 클릭하여 진입 |
| **최종 폴더** | **~mod** | **수동으로 생성** |

### 5단계: 모드 파일 전송
다운로드 폴더에서 압축을 푼 파일(`.pak`, `.ucas`, `.utoc` 파일)을 방금 만든 `~mod` 폴더로 직접 이동합니다. 이 파일들이 배치되면 게임은 기본 찰리 스킨 대신 이 파일들을 자동으로 로드합니다.

## 모드 테스트 및 확인

파일을 이동한 후, 프로세스가 작동했는지 확인할 차례입니다. 평소처럼 Steam을 통해 게임을 실행하세요. **스테이지 선택(Stage Select)** 화면으로 이동합니다. 설치가 성공했다면 찰리의 캐릭터 미리보기가 다운로드한 캐릭터 스킨으로 바뀌어 있을 것입니다.

여전히 캐릭터가 찰리로 표시된다면 다음 사항을 다시 확인하세요:
*   폴더 이름이 그냥 `mod`가 아니라 `~mod`(물결표 포함)인지 확인하세요.
*   `~mod` 폴더 안의 파일이 실제 `.pak` 파일인지, 아니면 다른 하위 폴더에 들어 있는지 확인하세요.
*   선택 메뉴에서 올바른 캐릭터를 보고 있는지 확인하세요.

> **💡 팁:** 특정 캐릭터에 대해 하나의 "교체" 모드만 활성화할 수 있습니다. 동일한 슬롯에 스파이더맨과 하츠네 미쿠를 모두 설치하려고 하면 게임이 충돌하거나 하나만 로드될 수 있습니다.

## 모드 제거 및 찰리 복구 방법

원래의 게임 미학으로 돌아가고 싶거나 모드로 인해 성능 문제가 발생하는 경우, 제거하는 방법은 설치하는 것보다 훨씬 간단합니다. 게임 전체를 다시 설치할 필요는 없습니다.

1.  `~mod` 폴더로 다시 이동합니다: `Dead as Disco > Lakota > Content > Paks > ~mod`.
2.  제거하려는 모드 파일을 선택합니다.
3.  마우스 오른쪽 버튼을 클릭하고 **삭제**를 선택합니다.
4.  또는 나중에 다시 사용하고 싶다면 파일을 바탕화면의 "백업" 폴더로 이동할 수도 있습니다.
5.  게임을 재시작합니다.

`~mod` 폴더가 비워지면 게임은 원래의 에셋으로 기본 설정됩니다. 이것이 핵심 게임 파일을 건드리지 않고 **Dead as Disco 모드 설치 방법** 워크플로우를 관리하는 가장 깔끔한 방법입니다.

## 일반적인 설치 오류 문제 해결

명확한 가이드가 있더라도 기술적인 문제가 발생할 수 있습니다. **Dead as Disco 모드 설치 방법**과 관련된 대부분의 문제는 파일 경로 오류나 버전 불일치에서 비롯됩니다.

*   **게임이 실행되지 않음:** 모드를 추가한 후 게임이 열리지 않으면 파일이 손상되었을 수 있습니다. `~mod` 폴더의 내용을 삭제하고 모드를 다시 다운로드해 보세요.
*   **캐릭터가 보이지 않음:** 이는 보통 `.ucas` 또는 `.utoc` 파일이 누락되었을 때 발생합니다. 다운로드한 세 개의 파일이 모두 폴더로 함께 이동되었는지 확인하세요.
*   **성능 끊김(Stuttering):** 폴리곤이 많은 스킨은 때때로 저사양 시스템에서 렉을 유발할 수 있습니다. 프레임 속도가 떨어지면 모드의 "라이트(lite)" 버전이 있는지 확인해 보세요.

## FAQ

**Q: Dead as Disco의 최고의 모드는 어디에서 찾을 수 있나요?**

A: 가장 신뢰할 수 있는 소스는 Nexus Mods입니다. 제작자들이 스킨, 스테이지, UI 개편 등을 업로드하는 게임 전용 카테고리가 마련되어 있습니다.

**Q: 모드를 설치하면 Steam 도전 과제가 비활성화되나요?**

A: 일반적으로 Dead as Disco의 스킨 수정은 순수하게 시각적인 요소이며 게임의 로직이나 난이도를 변경하지 않으므로 도전 과제를 비활성화하지 않습니다.

**Q: 콘솔 버전 게임에도 모드를 설치할 수 있나요?**

A: 아니요, 여기서 설명한 **Dead as Disco 모드 설치 방법**은 PC 버전 전용입니다. 콘솔은 일반적으로 사용자가 이러한 덮어쓰기에 필요한 파일 디렉토리 구조에 접근하는 것을 허용하지 않습니다.

**Q: 게임이 업데이트되어 모드가 작동하지 않으면 어떻게 해야 하나요?**

A: 게임이 업데이트되면 내부 파일 경로가 변경될 수 있습니다. Nexus Mods의 모드 페이지에서 업데이트된 버전이 있는지 확인해야 합니다. 업데이트가 없다면 `~mod` 폴더를 삭제하고 다시 생성하여 새로고침이 되는지 확인해 보세요.
